## 3. 피드백 루프

### 잘된 점
- 구현 전 기존 렌더링/스타일 흐름을 먼저 확인해 영향 범위를 정확히 제한했다.
- 기능 검증을 빌드 실행 + 단위 케이스 점검으로 분리해 회귀 가능성을 낮췄다.

### 문제·병목
- 최초 테스트 스크립트 문자열 이스케이프 이슈로 오탐이 발생했으나, 입력 문자열을 정리해 실제 로직 검증으로 복구했다.

### 레슨
- markdown-like renderer 확장 시에는 치환 순서뿐 아니라 치환 결과 HTML 구간 보호 전략이 함께 필요하다.

### 개선 액션
- 추후 renderer 기능 확장 시, 최소 샘플 테스트를 스크립트화해 재사용 가능한 회귀 점검 루틴으로 유지.
