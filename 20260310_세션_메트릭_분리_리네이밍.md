## 의사결정 과정

### Q1. `message_count` 처리 방식 (핵심 결정)

**논의 경과**:

처음에는 원본 계획대로 `message_count` 유지 + `user_entry_count` 신규 추가 vs alias를 제안했으나, 사용자가 "명시적 이름을 좋아한다"며 리네이밍 가능 여부를 질문했다. 이후 alias 방식의 장기 복잡도 문제를 확인하고, 소규모 DB + 1인 사용 환경에서는 완전 교체가 실익이 크다는 결론에 도달.

**검토한 옵션**:

| 옵션 | 내용 | 판단 |
|------|------|------|
| 유지 + 주석 | message_count 그대로, 의미만 주석으로 명시 | 가장 안전하나 이름 모호성 지속 |
| 리네이밍 + alias | DB는 user_entry_count, JS는 두 이름 노출 | ⚠️ 장기 복잡도 증가 — 채택하지 않음 |
| ✅ **완전 교체** | DB 컬럼 + JS 프로퍼티 + 소비자 4곳 일괄 변경 | alias 없이 깔끔. 소비자 4곳 단순 치환 |
| 양쪽 다 추가 | message_count 유지 + user_entry_count 신규 추가 | 동일 값 중복 컬럼 — 채택하지 않음 |

**결정**: `message_count` → `user_entry_count`로 **완전 교체** (alias 없음)

**근거**: 대형 DB·팀 프로젝트에서 리네이밍을 피하는 이유(수십 소비자, 배포 중 구/신 스키마 공존, 롤백 리스크)가 이 환경에는 해당 없음. 소비자 4개의 단순 치환이고, 이후 코드 가독성 이득이 큼.

### Q2. `tool_result_count` 카운트 단위

**결정**: block 수 (`findToolResults()`와 일관, 정보량 더 많음)

### Q3. 기존 DB 강제 재동기화

**결정**: mtime=0 리셋 — ALTER TABLE 시 자동 실행, 추가 코드 불필요

### Q4. `extract_session_gaps.js` 처리 범위

**결정**: 같은 세션에서 함께 수정 (git 커밋 scope는 분리)

## 구현 계획

### Context

`session-parser.js`의 `processSession()`에서 `entry.type === "user"`이면 무조건 `messageCount++`하여, `tool_result`만 담긴 user 엔트리도 "사용자 메시지"로 카운트된다. 예: 실제 사용자 입력 2개인 세션이 `message_count = 7`로 집계. `--skip-if-short` 기준과 대시보드 표시에 왜곡 발생.

### 확정된 의사결정

| 항목 | 결정 | 근거 |
|------|------|------|
| `message_count` 처리 | `user_entry_count`로 **완전 교체** (alias 없음) | 소규모 DB + 1인 사용, 소비자 4곳 단순 치환. 장기 복잡도 > 단기 호환성 |
| `tool_result_count` 단위 | **block 수** | `findToolResults()`와 일관, 정보량 더 많음 |
| 재동기화 방식 | **mtime=0 리셋** | ALTER TABLE 시 자동 실행, 추가 코드 불필요 |
| `extract_session_gaps.js` | **함께 수정** (레포 외부지만 같은 세션에서) | git 커밋 scope는 분리 |

### Step 1: session-parser.js — 메트릭 계산 로직

**파일**: `shared/session-parser.js`

**processSession() (줄 217~342)**:

줄 235 근처 카운터 변경:
```javascript
let userEntryCount = 0;       // 기존 messageCount 리네이밍
let userTextMessageCount = 0; // 실제 텍스트가 있는 user 엔트리
let toolResultCount = 0;      // tool_result block 수
```

줄 250-258 user 블록:
```javascript
if (entry.type === "user") {
  userEntryCount++;

  // tool_result block 카운트
  const content = entry.message?.content;
  if (Array.isArray(content)) {
    toolResultCount += content.filter(b => b.type === "tool_result").length;
  }

  // 실제 사용자 텍스트 판정
  const rawText = getTextFromMessage(entry.message);
  const cleanText = stripSystemTags(rawText);
  if (cleanText.trim()) {
    userTextMessageCount++;
  }

  messages.push({
    role: "user",
    text: cleanText,
    timestamp: entry.timestamp,
  });
}
```

줄 321-338 metadata:
- `messageCount` → `userEntryCount`
- `userTextMessageCount`, `toolResultCount` 추가

**processCodexSession() (줄 345~479)**:
- 줄 374 근처: `let toolResultCount = 0;` 추가
- response_item 처리에 `function_call_output` 분기 → `toolResultCount++`
- 줄 455: `messageCount` → `userEntryCount` (Codex는 이미 실제 user message만 카운트)
- metadata에 `userEntryCount`, `userTextMessageCount: userEntryCount`, `toolResultCount` 추가

**parsePlan() (줄 482~543)**:
- 줄 534: `messageCount: 0` → `userEntryCount: 0`, `userTextMessageCount: 0`, `toolResultCount: 0`

### Step 2: session-db.js — 스키마 마이그레이션 + 매핑

**파일**: `shared/session-db.js`

**_init()**: CREATE TABLE의 `message_count` → `user_entry_count`로 변경하고, 기존 DB 마이그레이션:
```javascript
const cols = this.db.prepare("PRAGMA table_info(sessions)").all().map(c => c.name);
if (cols.includes("message_count")) {
  this.db.exec("ALTER TABLE sessions RENAME COLUMN message_count TO user_entry_count");
  this.db.exec("ALTER TABLE sessions ADD COLUMN user_text_message_count INTEGER DEFAULT 0");
  this.db.exec("ALTER TABLE sessions ADD COLUMN tool_result_count INTEGER DEFAULT 0");
  this.db.exec("UPDATE sessions SET mtime = 0");
} else if (!cols.includes("user_text_message_count")) {
  this.db.exec("ALTER TABLE sessions ADD COLUMN user_text_message_count INTEGER DEFAULT 0");
  this.db.exec("ALTER TABLE sessions ADD COLUMN tool_result_count INTEGER DEFAULT 0");
}
```

**_upsertSession()**: `message_count` → `user_entry_count`, 신규 2개 컬럼 추가

**_rowToMeta()**:
```javascript
userEntryCount: row.user_entry_count || 0,
userTextMessageCount: row.user_text_message_count || 0,
toolResultCount: row.tool_result_count || 0,
```
