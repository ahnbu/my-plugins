# handoff 폴더명 변경 의사결정

날짜: 2026-02-28
결론: **현행 유지 (`handoff/`, `handoff_YYYYMMDD_NN_요약.md`)**

---

## 검토 안건

### 안건 1. `handoff` → `wrap` 으로 변경

**기각.**

- `handoff`는 파일의 **내용(다음 세션으로 넘기는 인수인계 문서)**을 설명하는 이름.
- `wrap`은 그것을 만드는 **행동(세션 마무리 커맨드)**의 이름. 폴더명은 행동이 아닌 내용물을 기술해야 함.
- `/wrap` 커맨드명과 일치시키려는 의도는 이해하나, `wrap/wrap_20260228_01_....md` 구조는 내용이 덜 명확해짐.
- 글로벌 CLAUDE.md, SKILL.md, 기존 문서 전반에 `handoff/`가 참조되고 있어 마이그레이션 비용 대비 실익 없음.

### 안건 2. `handoff` → `.handoff` (점 접두사)

**기각.**

- 점 접두사로 파일 탐색기 상단 정렬을 기대했으나, 이는 Linux/macOS 일부 환경의 관행.
- **Windows 탐색기**에서는 점 폴더를 알파벳 순으로 처리 → 상단 정렬 효과 없음.
- VS Code 파일 탐색기도 `explorer.sortOrder` 설정에 따라 숨기거나 하단 배치할 수 있음.
- `ls`에서 `-a` 없이는 보이지 않는 부작용 발생 가능.

---

## 결론

두 변경 모두 현재 환경(Windows)에서 실익이 없고, 기존 레퍼런스와의 불일치 비용이 더 큼.
`handoff/` 폴더명 및 `handoff_YYYYMMDD_NN_요약.md` 파일명 규칙을 **현행 유지**.
