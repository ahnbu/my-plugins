# Handoff 문서 템플릿

각 항목은 실제 내용이 있을 때만 포함하고, 해당 없는 항목은 생략한다.

```markdown
# Handoff — [작업명] (세션 NN)
> 날짜: YYYY-MM-DD
> 세션 ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
> 상태: 진행중 | 세션완료 | 전체완료

---

## 1. 현재 상태

### 작업 목표
- 이번 프로젝트의 최종 목표 1줄

### 진행 현황
| 단계 | 상태 | 산출물 |
|------|------|--------|
| (단계명) | ✅ 완료 / 🔄 진행중 / ⬜ 미착수 | (파일명 또는 -) |

### 핵심 의사결정 로그
- [결정 1] A 대신 B 채택. 이유: ~
- [결정 2] X는 스코프 제외. 이유: ~

### 다음 세션 시작점
- 구체적으로 어디서부터 이어가면 되는지 (파일명 + 위치 + 할 일)

---

## 2. 변경 내역 (이번 세션)
- 파일 단위로 추가/수정/삭제 사항을 충분히 기술
- git diff 기반으로 자동 수집 후 AI가 설명 보완

---

## 3. 피드백 루프
> ⚠️ 이 섹션은 AI 초안입니다. 검토·수정해 주세요.

### 잘된 점
- 효과적이었던 접근 방식, 도구, 프로세스

### 문제·병목
- 막혔던 지점, 비효율, 예상과 달랐던 것

### 레슨 (재사용 가능한 교훈)
- "[맥락]에서는 [방법]이 효과적/비효과적" 형식

### 개선 액션
- 도출된 구체적 행동 변경
- 적용 범위: 이 프로젝트 한정 | 전역 지침 반영 | 개인 맞춤설정 반영
```
