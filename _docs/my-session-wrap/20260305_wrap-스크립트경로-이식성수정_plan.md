# Plan: my-session-wrap SKILL.md 스크립트 경로를 이식 가능하게 수정

> 저장일: 2026-03-05 | 원본: `C:\Users\ahnbu\.claude\plans\calm-yawning-sunrise.md`

## 개요

`/wrap` 스킬의 `next-handoff.sh` 스크립트 경로가 실행 컨텍스트에서 해석 불가하여 AI가 스크립트를 우회하고 수동으로 handoff 파일명을 생성 → 날짜 오류 발생. 이를 bash fallback 패턴으로 수정하여 스크립트가 항상 호출되도록 한다.

---

## 사용자 문제 상황

### 발생 경위
1. `/wrap` 실행 시 AI가 handoff 파일명을 생성해야 함
2. SKILL.md에 `$(dirname "$0")/scripts/next-handoff.sh`로 스크립트 경로가 명시됨
3. SKILL.md는 bash 스크립트가 아닌 **마크다운 프롬프트** → `$0`이 동작하지 않음
4. AI가 스크립트 실행을 포기하고 수동으로 파일명 생성
5. 기존 handoff 파일 목록(`ls _handoff/ | grep "20260304"`)에서 날짜를 앵커링
6. **오늘 날짜(20260305)가 아닌 어제 날짜(20260304)로 handoff 생성** → 날짜 오류

### 핵심 문제
- 스크립트(`next-handoff.sh`)는 `date +%Y%m%d`로 날짜를 자동 계산하므로 정상 동작
- 스크립트가 호출되지 않은 것이 근본 원인
- 이전 세션에서 `$HOME/.claude/my-claude-plugins/...`로 하드코딩 수정했으나, 이는 marketplace 배포 시 캐시 경로와 달라 이식성 문제 발생

### 사용자 피드백
- "Glob 도구 호출은 느리지 않나?" → Glob 도구 대신 bash 셸 내 glob 확장 방식 채택
- "배포해서 플러그인 업데이트하면 달라지는 거 아냐?" → 로컬 dev + 캐시 두 경로 모두 커버 필요

## Context

`/wrap` 실행 시 handoff 파일명을 결정하는 `next-handoff.sh` 스크립트 호출이 실패하여
AI가 스크립트를 건너뛰고 수동으로 파일명을 생성 → 날짜 오류 발생.

### 근본 원인
- SKILL.md는 마크다운 프롬프트 — `$0`, `${CLAUDE_PLUGIN_ROOT}` 모두 미작동
- 기존 경로 `$(dirname "$0")/scripts/...`가 실행 불가 → AI가 스크립트를 포기하고 수동 생성
- 스크립트 자체는 정상 (`date +%Y%m%d`로 날짜 자동 결정)

### 접근 방식 선택

| 방식 | 속도 | 이식성 | 복잡도 |
|------|------|--------|--------|
| A. Glob 도구로 탐색 | 느림 (도구 호출) | 높음 | 중간 |
| **B. bash fallback (ls + glob)** | 즉시 (셸 내 처리) | 높음 | 낮음 |
| C. 하드코딩 절대경로 | 즉시 | 낮음 | 낮음 |

**→ B 채택**: bash 한 줄로 로컬 dev/캐시 두 경로를 모두 검색. 셸 glob 확장이므로 즉시 완료. 도구 호출 오버헤드 없음.

## 변경 내용

Step 2-2의 스크립트 호출부를 bash fallback 패턴으로 교체:

**변경 전:**
```bash
bash "$HOME/.claude/my-claude-plugins/my-session-wrap/skills/my-session-wrap/scripts/next-handoff.sh" "" "<요약>"
```

**변경 후:**
```bash
SCRIPT=$(ls "$HOME/.claude/my-claude-plugins/my-session-wrap/skills/my-session-wrap/scripts/next-handoff.sh" "$HOME/.claude/plugins/cache/my-claude-plugins/my-session-wrap"/*/skills/my-session-wrap/scripts/next-handoff.sh 2>/dev/null | head -1)
bash "$SCRIPT" "" "<요약>"
```

**동작**: 로컬 개발 경로 → 캐시 경로(전 버전) 순으로 탐색, 첫 hit 사용. `ls`+glob이므로 즉시.

## 보충: 대화 중 검토 내용

### `${CLAUDE_PLUGIN_ROOT}` 제한 사항 (탐색 결과)

- hooks.json, .mcp.json에서는 정상 확장 (Claude Code가 처리)
- SKILL.md(마크다운 프롬프트)에서는 **확장되지 않음** (GitHub #9354)
- 공식 plugin-dev 문서는 SKILL.md에서도 사용 가능하다고 안내하나, 실제로는 미작동

### 플러그인 설치 경로 구조

| 환경 | 경로 |
|------|------|
| 로컬 개발 | `~/.claude/my-claude-plugins/my-session-wrap/skills/.../scripts/` |
| 캐시 (marketplace) | `~/.claude/plugins/cache/my-claude-plugins/my-session-wrap/<version>/skills/.../scripts/` |

### 검토하고 제외한 대안

- **Glob 도구 탐색 (방식 A)**: 동작은 확실하나 도구 호출 오버헤드가 추가되어 `/wrap`이 느려지므로 사용자가 거부.
- **하드코딩 절대경로 (방식 C)**: 로컬 개발에서만 동작하고 캐시 경로에서는 실패하므로 이식성 부족.
- **Hook으로 경로 주입**: 정확하지만 hook 추가 + 파일 I/O 복잡도가 과도.
- **스크립트 로직 인라인**: SKILL.md가 비대해지고 스크립트와 이중 관리 필요.
