### serve.js 동작 흐름

1. `server.listen(0)` — OS 자동 포트 할당
2. 브라우저 오픈: `start "" "http://127.0.0.1:{port}"`
3. 기존 `output/index.html` 있으면 즉시 서빙 (프로그레스바 + SSE 주입)
4. 없으면 "빌드 중..." 로딩 페이지 표시
5. 800ms 후 `build.js` 스폰
6. stdout 파싱 → SSE 진행률 전송
7. 빌드 완료 → `event: reload` 전송 → 브라우저 자동 리로드
8. 리로드 후 SSE 재연결 → 서버가 `no-update` 즉시 전송 → 오버레이 제거
9. 5초 후 서버 자동 종료

### 프로그레스 오버레이 HTML (body 태그 직후 주입)

```html
<div id="build-progress" style="position:fixed;top:0;left:0;right:0;z-index:9999;...">
  <div id="bp-bar" style="background:#8b5cf6;..."></div>
  <span id="bp-text">업데이트 확인 중...</span>
</div>
<script>
const es = new EventSource('/events');
es.addEventListener('progress', e => { bar.style.width = e.data + '%'; });
es.addEventListener('reload', () => { es.close(); location.reload(); });
es.addEventListener('no-update', () => { overlay.remove(); es.close(); });
</script>
```

## 2단계: 버그 발견 및 부분 수정

### 버그 1 — OUTPUT_HTML 경로 불일치 (확정)

실행 결과: "기존 HTML 즉시 로딩이 되어야 하는데, 빌드중 페이지 표시"

```
build.js 실제 출력: my-claude-plugins/output/session-dashboard/index.html
                                    ↑ __dirname 기준 "../output/session-dashboard"

serve.js 참조 (초기 오류): my-session-dashboard/output/session-dashboard/index.html
                                                  ↑ __dirname 기준 "output/session-dashboard"

serve.js 1차 수정: my-claude-plugins/output/session-dashboard/index.html  (경로 수정)
```

### 버그 2 — SSE ERR_CONNECTION_REFUSED (의심)

```
:55922/events:1  Failed to load resource: net::ERR_CONNECTION_REFUSED  (반복)
```

가능한 원인: Windows에서 `localhost` → `::1`(IPv6) 해석 시, `127.0.0.1`(IPv4)만 리슨하는 서버에 연결 불가

수정 내용:
- `server.listen(0, '127.0.0.1')` → `server.listen(0)` (모든 인터페이스)
- 브라우저 열기 URL: `localhost` → `http://127.0.0.1:{port}` (명시적 IPv4)
- `exec('start http://...')` → `exec('start "" "http://..."')` (URL 모호성 방지)
- `process.on('uncaughtException', ...)` 추가
