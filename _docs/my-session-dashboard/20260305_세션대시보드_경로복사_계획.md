## Context

세션 대시보드의 detail-header 영역에 원본 파일 경로가 표시되지 않아, 사용자가 plan/session 파일 위치를 알 수 없다. 파일 경로 표시 + 클립보드 복사, 문서 전체 내용 복사 기능을 추가한다.

## 현황

- **build.js**: 세션/Plan 파일의 절대 경로를 알고 있지만 metadata에 저장하지 않음
- **index.html**: clipboard/copy 관련 코드 없음
- Plan 상세: `showPlanDetail()` (772행), 세션 상세: `selectSession()` (698행)

## 변경 파일

### 1. `build.js` — metadata에 `filePath` 추가

- **세션** (`processSession`, 300행): `filePath` 파라미터를 받아 metadata에 추가
- **호출부** (515행): `processSession(filePath)` → `filePath` 전달 (이미 변수 존재)
- **캐시 히트** (504행): `cached.metadata`에도 `filePath` 포함되도록 처리
- **Plan** (`parsePlan`, 371행): `filePath` 파라미터를 받아 metadata에 추가
- Plan 호출부 (430행): 이미 `filePath` 변수 존재, metadata에 추가만 하면 됨

### 2. `index.html` — UI + JS 변경

#### CSS 추가
```css
.copy-btn { ... }         /* 경로 옆 복사 아이콘 버튼 */
.copy-btn-full { ... }    /* "문서 복사" 텍스트 버튼 */
.copy-toast { ... }       /* 복사 완료 알림 */
.file-path-row { ... }    /* 경로 표시 행 */
```

#### JS 변경

- `copyToClipboard(text)` 헬퍼 함수 추가 (navigator.clipboard.writeText + fallback)
- `copyToast()` 함수 — 복사 완료 시 토스트 표시

- **`showPlanDetail()`**: detail-meta 아래에:
  - 파일 경로 행 + 복사 버튼
  - "문서 전체 복사" 버튼 (plan의 raw markdown content를 클립보드에 복사)

- **`selectSession()`**: detail-meta 아래에:
  - 파일 경로 행 + 복사 버튼
  - (세션은 JSONL이므로 "문서 전체 복사"는 의미 낮음 → Plan에만 적용)

### 3. `index.html` — 헤더 레이아웃 변경

현재: `[Claude 세션] [타입필터] [통계] .............. [검색]`
변경: `[검색] [전체/세션/플랜] [통계]`

- "Claude 세션" h1 텍스트 제거
- 검색 박스를 가장 왼쪽으로 이동 (`margin-left: auto` 제거)
- 타입 필터 → 검색 박스 오른쪽
- 통계 → 타입 필터 오른쪽

CSS 변경:
- `.search-box`에서 `margin-left: auto` 제거, `flex` 유지
- `.header .stats`에 `margin-left: auto` 추가 (우측 정렬)

HTML 변경 (순서):
```html
<div class="header">
  <div class="search-box">...</div>
  <div class="type-filters" id="typeFilters"></div>
  <span class="stats" id="stats"></span>
</div>
```
