## 5. 주요 인사이트

### [A] 메타작업 비중이 70%+

실제 프로덕트 개발보다 **Claude Code 환경 자체를 개선하는 워크플로우 인프라 정비** 작업이 대부분. 플러그인, CLAUDE.md, MCP 설정, 세션 관리 도구 등.

### [B] 플러그인 시스템 안정성이 최대 고통점

Unknown skill, EEXIST, 커맨드 미등록, 로컬 vs 마켓플레이스 혼동이 가장 많은 시간을 소비. ensure-commands.js 신뢰도 낮아 매 세션 수동 확인 필요.

### [C] Claude 지시 불이행 불만

CLAUDE.md에 명시한 규칙(fetch MCP 사용 등)을 Claude가 따르지 않는 문제가 반복. 서브에이전트 프롬프트에 명시하는 워크어라운드로 대응.

### [D] 컨텍스트 소진이 만성적 제약

장시간 탐구 세션에서 컨텍스트 소진이 반복. handoff 기반 세션 이어가기 패턴이 이 문제의 대응책으로 발전.

### [E] 탐색 후 포기 사례

- 음성입력: WhisperWriter/VoiceMode MCP → 최종 포기, Win+H 채택
- Notion MCP: 시행착오 많아 아직 미정착

### [F] 문서화 철학 발전 중

02-25에 AI 에이전트 시대의 문서관리 방법론에 대한 심층 탐구 시작. context.md vs handoff.md vs CHANGELOG.md vs README.md 역할 구분 논의.
